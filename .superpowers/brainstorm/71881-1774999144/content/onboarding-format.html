Format de l'onboarding pro

Après inscription, comment guider le pro pour configurer son salon ?

A

Wizard étape par étape

Un assistant guidé en 4-5 étapes séquentielles : nom du salon → catégories → premier soin → horaires → publication. Le pro ne peut pas sauter d'étapes. Progression claire avec barre d'avancement.

Avantages

  • Impossible de rater une étape
  • Expérience fluide et rassurante
  • Conversion maximale

Inconvénients

  • Rigide — frustrant si le pro sait déjà ce qu'il fait
  • Plus de code à écrire (composant dédié)
B

Dashboard avec checklist

Le dashboard affiche une checklist de configuration (✓ Nom du salon, ✓ Catégories, ○ Horaires...). Le pro clique sur chaque item pour aller à la page correspondante. Ordre libre.

Avantages

  • Flexible — le pro fait dans l'ordre qu'il veut
  • Réutilise les pages existantes (/pro/salon, /pro/categories...)
  • Moins de code nouveau

Inconvénients

  • Moins guidé — risque d'abandon
  • Le pro peut publier avec un salon incomplet
C

Hybride : checklist + bloquant

Dashboard avec checklist comme B, mais le bouton "Publier mon salon" est désactivé tant que les étapes obligatoires ne sont pas complétées. Première visite : une bannière suggère de commencer par le nom du salon.

Avantages

  • Flexible ET sécurisé
  • Réutilise les pages existantes
  • Bloque la publication si incomplet

Inconvénients

  • Validation de complétude à implémenter côté backend

💡 Je recommande l'option C — elle offre la flexibilité de B avec la sécurité de A, et réutilise les pages pro existantes.
