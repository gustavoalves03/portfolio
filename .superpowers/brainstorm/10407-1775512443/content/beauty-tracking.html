4. Architecture données

Les entités backend nécessaires

ClientProfile (tenant-scoped)
├── id, userId, notes, skinType, hairType
├── allergies, preferences
├── consentPhotos (boolean)
├── consentPublicShare (boolean)
└── consentGivenAt (timestamp)

VisitRecord (tenant-scoped)
├── id, clientProfileId, bookingId
├── careId, careName
├── visitDate, practitionerNotes
├── productsUsed (JSON list)
├── satisfactionScore (1-5, nullable)
├── satisfactionComment
└── createdAt

VisitPhoto (tenant-scoped)
├── id, visitRecordId
├── photoType (BEFORE / AFTER / PROGRESS)
├── imagePath
├── imageOrder
└── createdAt

ClientReminder (shared schema)
├── id, userId, tenantSlug
├── careId, careName
├── recommendedDate
├── message
├── sent (boolean)
└── createdAt

Liens existants :
- CareBooking.userId → User (shared)
- VisitRecord.bookingId → CareBooking (tenant)
- VisitPhoto → uploads/visits/{visitId}/
- Post.beforeAfter → peut être lié à VisitRecord
